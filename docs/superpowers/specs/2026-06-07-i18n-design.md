# 앱 다국어화 (i18n) 설계

## 개요

react-i18next를 사용해 앱 전체 UI 텍스트를 한국어/영어로 전환할 수 있도록 한다. 언어 설정은 localStorage에 저장되며 사이드바의 KO/EN 토글 버튼으로 전환한다.

---

## 범위

**대상 언어:** 한국어(ko), 영어(en)  
**기본 언어:** 한국어  
**대상 문자열:** 앱 UI 텍스트 약 76개 (버튼, 라벨, 빈 상태 안내, 에러 메시지)  
**범위 외:** 뉴스 기사 콘텐츠 번역 (2단계에서 별도 구현)

## i18next 설정

```js
// src/i18n.js
import i18n from 'i18next'
import { initReactI18next } from 'react-i18next'
import ko from './locales/ko.json'
import en from './locales/en.json'

i18n.use(initReactI18next).init({
  resources: { ko: { translation: ko }, en: { translation: en } },
  lng: localStorage.getItem('i18nextLng') ?? 'ko',
  fallbackLng: 'ko',
  interpolation: { escapeValue: false },
})

i18n.on('languageChanged', (lng) => localStorage.setItem('i18nextLng', lng))

export default i18n
```

## 언어 토글 UI

사이드바 하단에 KO / EN 토글 버튼 추가:

```
┌─────────────────┐
│  대시보드        │
│  캘린더          │
│  뉴스           │
│                 │
│  [KO]  [EN]     │  ← 사이드바 하단
└─────────────────┘
```

- 현재 선택된 언어 버튼은 활성 스타일 (기존 `.type-btn.active` 패턴 사용)
- `i18n.changeLanguage('en')` 호출로 전환

## 동적 문자열 처리

### `Header.jsx` — 상대 시간

```js
// 현재
`방금 업데이트` / `${mins}분 전` / `${hrs}시간 전`

// 변경 후
t('header.justUpdated')
t('common.minutesAgo', { count: mins })
t('common.hoursAgo', { count: hrs })
```

### `src/utils/finnhub.js` — `formatPublishedAt`

React 외부이므로 `i18n.t()` 직접 사용. `common.hoursAgo` 키 재사용:

```js
import i18n from '../i18n.js'

export function formatPublishedAt(unixTs) {
  if (!unixTs) return ''
  const diffHours = Math.floor((Date.now() - unixTs * 1000) / (1000 * 60 * 60))
  if (diffHours < 24) return i18n.t('common.hoursAgo', { count: diffHours })
  const d = new Date(unixTs * 1000)
  return `${d.getFullYear()}-${String(d.getMonth() + 1).padStart(2, '0')}-${String(d.getDate()).padStart(2, '0')}`
}
```

### `src/hooks/useStockNews.js` — 에러 메시지

```js
import i18n from '../i18n.js'

// 기존
setError('뉴스를 불러올 수 없습니다')

// 변경 후
setError(i18n.t('news.error'))
```

언어 토글 버튼은 기존 `.type-btn` / `.type-btn.active` 스타일 재사용. 추가 CSS 불필요.

## 테스트 전략

- `formatPublishedAt` 유닛 테스트: 기존 테스트가 `"2시간 전"` 문자열을 직접 비교함. i18n 도입 후 `i18n.t`가 올바른 값을 반환하려면 테스트 파일에서 `import i18n from '../i18n.js'` 후 `i18n.changeLanguage('ko')`를 `beforeAll`에서 호출해야 함. `i18n.js`는 `localStorage` 없는 Node 환경에서도 `lng: 'ko'`로 초기화됨.
- 나머지 컴포넌트: UI는 브라우저에서 KO↔EN 전환 후 각 페이지 텍스트 변경 확인

## 범위 외

- 뉴스 기사 콘텐츠 번역 (2단계)
- 복수형(plural) 처리 (해당 문자열 없음)
- 날짜 포맷 현지화
- 3번째 언어 추가
